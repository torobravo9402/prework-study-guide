HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Use HTML to build a sturdy frame, using HTML elements as our frame’s building blocks.
  • HTML attributes are always defined in the beginning element, never the closing element.
  • An HTML developer is responsible for not only providing the structure for the webpage but also making sure that it provides a positive user experience.
  • Inspect the parent < body > element and its child elements, and reverse engineer the code. In doing so, you will analyze how they structure the webpage's content.
  • The < body > element is an example of a semantic HTML element.
  • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage.
  • Use semantic HTML elements to indicate the role of their content, as well as to give structure to the content of the webpage.
  • The < ul >, < ol >, and < li > elements represent unordered lists, ordered lists, and list items. In essence, these HTML elements represent bulleted or numbered lists.
  • The `src` attribute defines the location, or source, of the image file.
  • The `alt` attribute contains a text string that describes the appearance and functionality of the image.

CSS

  • Need to create an external style sheet with CSS.
  • CSS determines the styles of a webpage in order to make a website visually appealing. CSS can also make a website simpler to use with a polished design and consistent organization.
  • Separation of concerns is a widely used term in web development and refers to separating code into different files based on use. It is one of the key design principles that we practice in the boot camp.
  • Patience and a systematic approach are key to successfully identifying and fixing bugs in your code.
  • Link the style sheet to the HTML.
  • External style sheet, added CSS, and connected the style sheet to the HTML document.
  • used HTML elements such as < header >, < footer >, and < h1 > as selectors in the external style sheet and then applied CSS properties to style them.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector. Then the class attribute can be assigned to the element in the HTML, enabling us to freely choose which elements will be assigned the CSS rule.
  • Use the class selector, card, to style each topic in our Prework Study Guide.
  • The .card selector contains width, margin, border, and box-shadow properties.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

JavaScript

  • Examined basic JavaScript features, such as variables, strings, and data types.
  • Browser Application Programming Interfaces (APIs) built into web browsers, providing functionality such as dynamically creating HTML and setting CSS styles; collecting and manipulating a video stream from a user's webcam, or generating 3D graphics and audio samples.
  • In JavaScript, instructions are called statements and are separated by semicolons (;).
  • It is considered best practice, however, to always write a semicolon after a statement, even when it is not strictly needed. This practice reduces the chances of bugs getting into the code.
  • Declarations: JavaScript has three kinds of variable declarations: var, let, const.
  • var: Declares a variable, optionally initializing it to a value.
  • let: Declares a block-scoped, local variable, optionally initializing it to a value.
  • const: Declares a block-scoped, read-only named constant.
  • Variables: You use variables as symbolic names for values in your application. The names of variables, called identifiers, conform to certain rules.
  • A JavaScript identifier usually starts with a letter, underscore (_), or dollar sign ($). Subsequent characters can also be digits (0 – 9). Because JavaScript is case sensitive, letters include the characters A through Z (uppercase) as well as a through z (lowercase).
  • You can use most Unicode letters such as å and ü in identifiers.
